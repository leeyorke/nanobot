# Kanban格式规范

本项目的Kanban文件完全兼容Obsidian Kanban插件格式，可以直接在Obsidian中安装Kanban插件后打开使用，支持拖拽修改任务状态、添加新任务、修改任务详情等操作。

## 文件格式说明
`📋 kanban.md`文件包含两部分：
1. YAML frontmatter：标识这是一个Kanban文件
2. Markdown内容：按二级标题分组的任务列表

## 标准格式示例
```markdown
---
kanban-plugin: basic
---

## ⚡ 立即做

- [ ] 回复紧急客户邮件 ⏰ 2026-04-07 11:30
- [ ] 修复线上支付bug

## 🚶 下一步行动

- [ ] 写项目需求文档初稿
- [ ] 联系供应商确认报价

## 📅 日程

- [ ] 明天下午3点开需求评审会 ⏰ 2026-04-08 15:00
- [ ] 周五晚上参加朋友聚会 ⏰ 2026-04-11 19:00

## ⏳ 等待中

- [ ] 等待产品经理确认需求
- [ ] 等待开发环境部署完成

## 📦 项目

- [ ] 开发用户管理模块
- [ ] 组织Q2部门团建

## 🔮 将来做

- [ ] 学习Go语言
- [ ] 去西藏旅游

## ✅ 已完成

- [x] 完成上周周报提交 ✅ 2026-04-06 18:30
- [x] 服务器性能优化 ✅ 2026-04-07 10:00
```

## 格式要求

看板模板可见: [kanban_template](../assets/kanban_template.md)

### 1. YAML frontmatter
必须包含在文件最开头，格式为：
```yaml
---
kanban-plugin: board
---
```
Obsidian Kanban插件会根据这个标识识别这是Kanban文件。

### 2. 列标题
使用二级标题（`##`）作为Kanban的列名，列名必须和GTD分类一一对应：
- `## ⚡ 立即做` → 对应`02-Tasks/Now/`
- `## 🚶 下一步行动` → 对应`02-Tasks/Next/`
- `## 📅 日程` → 对应`02-Tasks/Calendar/`
- `## ⏳ 等待中` → 对应`02-Tasks/Waiting/`
- `## 📦 项目` → 对应`02-Tasks/Projects/`
- `## 🔮 将来做` → 对应`02-Tasks/Someday/`
- `## ✅ 已完成` → 对应`02-Tasks/Completed/`

### 3. 任务项格式
每个任务使用Markdown任务列表格式：
- 待完成：`- [ ] 任务内容`
- 已完成：`- [x] 任务内容`

### 4. 附加信息格式
#### 时间标记
使用`⏰ @{YYYY-MM-DD} HH:MM`格式标记任务截止时间：
```
- [ ] 明天下午3点开需求评审会 ⏰ @{2026-04-08} 15:00
```

#### 完成时间标记
使用`✅ YYYY-MM-DD HH:MM`格式标记任务完成时间：
```
- [x] 完成上周周报提交 ✅ @{2026-04-06} 18:30
```

#### 优先级标记
使用`🔴`（高）、`🟡`（中）、`🟢`（低）标记优先级：
```
- [ ] 修复线上支付bug 🔴
```

## 双向同步机制
1. **系统修改同步到Kanban**：当系统自动分类、修改任务状态、添加新任务时，会自动更新Kanban文件内容
2. **用户修改Kanban同步到系统**：系统会定期扫描Kanban文件的修改，当用户在Obsidian中修改任务状态、移动任务列、修改任务内容时，会自动同步到对应的文件目录和元数据
3. **冲突处理**：当两边同时修改时，以最新的修改时间为准，不会覆盖用户的手动修改

## Obsidian Kanban插件配置建议
1. 安装Kanban插件：在Obsidian社区插件中搜索"Kanban"安装
2. 打开设置：
   - 开启"Show task counts in column headers"（在列标题显示任务数量）
   - 开启"Show relative dates"（显示相对时间）
   - 开启"Automatically add completed date"（自动添加完成日期）
3. 使用技巧：
   - 拖拽任务卡片可以在不同列之间移动，自动修改任务分类
   - 点击任务卡片可以编辑任务详情
   - 勾选复选框可以标记任务完成，自动移动到"已完成"列
