# GTD工作流说明

## GTD核心思想
GTD（Getting Things Done）是一套高效的任务管理方法，核心是将所有事务从大脑中清空，通过系统化的分类和处理，让你专注于当前最重要的事情，减少焦虑和遗漏。

## 工作流完整流程
```
想法记录 → 收件箱（Inbox）→ 自动分类 → 任务归类 → 执行 → 回顾 → 归档
```

## 分类规则说明
### 1. ⚡ 立即做（Now）
**定义**：2小时以内可以完成的紧急且重要的任务
**判断标准**：
- 优先级为高
- 预计完成时间 ≤ 2小时
- 今天内必须完成
**示例**：
- 回复紧急客户邮件
- 修复线上紧急bug
- 提交今天截止的报告

### 2. 🚶 下一步行动（Next）
**定义**：有明确下一步动作、可以马上开始执行的任务
**判断标准**：
- 有清晰的下一步动作
- 没有明确的时间限制，但近期需要完成
- 不需要等待其他人或条件
**示例**：
- 写项目需求文档初稿
- 联系供应商确认报价
- 整理会议纪要

### 3. 📅 日程（Calendar）
**定义**：有明确的日期和时间安排的任务、活动、会议
**判断标准**：
- 包含明确的时间点/时间段
- 需要在特定时间出席或完成
**示例**：
- 明天下午3点开需求评审会
- 周五晚上参加朋友聚会
- 下周三出差去上海

### 4. ⏳ 等待中（Waiting）
**定义**：需要等待他人反馈或外部条件成熟才能继续的任务
**判断标准**：
- 依赖他人的工作成果
- 需要等待某个事件发生
- 正在等待回复/审批/资源
**示例**：
- 等待产品经理确认需求
- 等待开发环境部署完成
- 等待客户合同签署

### 5. 📦 项目（Projects）
**定义**：需要多个步骤、多天才能完成的大型任务
**判断标准**：
- 需要拆分成多个子任务
- 完成周期 > 1天
- 有明确的项目目标
**示例**：
- 开发新功能模块
- 组织部门年会
- 准备Q2季度汇报

### 6. 🔮 将来做（Someday）
**定义**：未来某一天可能要做，但目前没有明确时间安排的想法或任务
**判断标准**：
- 没有明确的截止时间
- 现在不需要立即处理
- 未来可能会做
**示例**：
- 学习Go语言
- 去西藏旅游
- 写一本技术书籍

## 自动分类流程
1. **新记录进入收件箱**：用户记录的所有内容首先进入`01-Inbox/`目录
2. **内容类型识别**：自动判断内容是待办、笔记、链接、活动还是资源
3. **待办筛选**：识别为待办的内容移动到`02-Tasks/Pending/`目录待分类
4. **GTD分类**：根据内容、时间、优先级等信息自动归类到对应的GTD目录
5. **看板同步**：自动更新`📋 kanban.md`文件，保持与目录结构同步
6. **不确定处理**：分类置信度低于80%时，主动询问用户确认分类

## 状态流转
```
unclassified → pending → [Now/Next/Calendar/Waiting/Projects/Someday] → completed → archived
                                                               ↓
                                                             deleted
```

## 回顾机制
### 每日回顾（21:00）
- 统计今日完成情况
- 处理未完成任务（延期/立即做/删除）
- 生成明日待办清单

### 每3天回顾
- 检查长期任务（等待中/项目/将来做）
- 确认是否需要调整优先级或分类
- 清理过时不需要的任务

### 每周回顾（周日20:00）
- 全面回顾本周所有任务和记录
- 整理收件箱未处理内容
- 规划下周重点任务
- 归档已完成的项目

### 每月回顾（每月1日）
- 上月数据归档备份
- 统计月度完成情况
- 调整下月目标和计划
